## Bug Fix 1 — Sons sumiram

**Causa**: `playTone` chamava `ctx.resume()` (async) e em seguida chamava `osc.start()` imediatamente, sem esperar a retomada do contexto. Se o AudioContext estava suspenso (inatividade > ~1min), o som não tocava.

**Fix**: `ctx.resume().then(play)` — o som só é iniciado APÓS o contexto estar em estado 'running'.

```js
// Antes (quebrado)
if(ctx.state==='suspended')ctx.resume();
const osc=ctx.createOscillator(); osc.start();

// Depois (correto)
const play=()=>{ const osc=...; osc.start(); };
if(ctx.state==='suspended') ctx.resume().then(play);
else play();
```

## Bug Fix 2 — Nav sem feedback visual imediato

**Causa**: A atualização do tab ativo dependia do IntersectionObserver (async, dispara quando 25% da seção entra em view). Clicar num tab não atualizava o indicador imediatamente.

**Fix**: Adicionado ao click handler do tab:
```js
document.querySelectorAll('.tab').forEach(t=>t.classList.remove('active'));
tab.classList.add('active');
moveSlider(tab);
```
Agora o clique: (1) feedback visual instantâneo → (2) scroll suave → (3) IntersectionObserver confirma quando a seção está visível.

## Bug Fix 3 — Canvas não preenchia a sessão 1

**Causa**: CSS `clamp(300px, ..., 580px)` limitava o canvas a 580px enquanto a viewport é ~1080px. O grid `flex-column` com `order: -1` também estava incompleto.

**Fix**: Layout grid 2 linhas para `#v-semente .canvas-layout`:
- Linha 1: `.stage` ocupa 100% da largura, `calc(100svh - 120px)` de altura (viewport menos header+nav)
- Linha 2: `.sidebar-left` e `.sidebar-right` lado a lado

```css
#v-semente .canvas-layout {
  display: grid;
  grid-template-rows: calc(100svh - 120px) auto;
  grid-template-columns: 1fr 1fr;
}
#v-semente .stage { grid-column: 1/-1; grid-row: 1; }
#v-semente canvas {
  width: min(calc(100svh - 142px), 88vw);
  height: min(calc(100svh - 142px), 88vw);
}
```

Em 1080p: canvas é `min(940px, ~1690px) = 940px`. Cada pixel art pixel = 940/48 = **19.6px** — grande e nítido.
Em mobile 375px: canvas é `min(527px, 330px) = 330px`.

As sidebars ficam na linha 2 (scroll mínimo para ver), mantendo o conteúdo acessível.

---

## Decisões arquiteturais

**Por que grid e não flex para o hero layout?**
Com `grid-template-rows`, posso definir exatamente qual altura a primeira linha ocupa (calc com svh). Com flex, dependo do `flex-grow` que distribui espaço proporcional — menos previsível e mais difícil de depurar.

**Por que `min(calc(100svh - X), 88vw)` e não `calc` diretamente?**
O canvas deve caber tanto na altura (sem overflow vertical) quanto na largura (em telas estreitas/mobile). `min()` garante que o menor dos dois limites vença — sempre quadrado e sempre visível.
