## Fase 2 — CRESCER: cotilédones + caule

**`CANVAS_STAGE = 3 → 4`**

Adicionado `if (stage >= 4)` em `drawSprout()`:

```javascript
const sw = 0.7 + Math.sin(t * 0.018) * 0.12; // balançar suave
// Caule
px(cx, 25, P.stem, 0.92); px(cx, 24, P.stem, 0.85);
// Cotilédone esquerdo (leque)
px(cx-1,25,P.leaf,sw*0.95); px(cx-2,25,P.leaf,sw*0.80);
px(cx-2,24,P.leaf,sw*0.65); px(cx-3,24,P.leaf,sw*0.40);
// Cotilédone direito (simétrico)
px(cx+1,25,P.leaf,sw*0.95); px(cx+2,25,P.leaf,sw*0.80);
px(cx+2,24,P.leaf,sw*0.65); px(cx+3,24,P.leaf,sw*0.40);
// Nervura central
px(cx, 23, P.sprout, sw*0.50);
```

**Por que `sw = 0.7 + sin(t*0.018)*0.12`?**
Os cotilédones balançam suavemente — simulando respiração/fotossíntese. A frequência `0.018` é mais lenta que o pulso da semente (`0.025`) — cada elemento tem seu próprio ritmo de vida. A amplitude `0.12` é discreta: balançar, não sacudir.

**Por que cotilédones assimétricos (fade em leque)?**
A opacidade diminui do centro para as pontas: `0.95 → 0.80 → 0.65 → 0.40`. Isso cria ilusão de volume — a borda da folha é mais translúcida que o centro, como folhas reais contra a luz.

## Fase 4 — REVISAR: decisão sobre src/state.json

**Proposta original:** extrair `CANVAS_STAGE` para `src/state.json` para persistência entre sessões.

**Decisão: NÃO fazer.**

**Motivo:** `CANVAS_STAGE` não é estado de runtime — é estado arquitetural. Cada mudança de stage representa uma decisão deliberada de crescimento, documentada em um commit. **Git IS a persistência.** Extrair para JSON adicionaria:
- Uma fetch request a cada carregamento
- Complexidade de sincronização (o JSON precisaria ser atualizado junto com o código)
- Um novo tipo de arquivo sem versionamento semântico

**Alternativa adotada:** documentar explicitamente no código com o comentário:
```javascript
// CANVAS_STAGE — atualizado a cada /crescer-canvas
// Mudar = commit = decisão documentada. Git é a persistência.
```

Isso é um exemplo exato do tipo de decisão que a fase REVISAR deve produzir: às vezes a revisão resulta em "não mudar" — e documentar o porquê é igualmente valioso.
