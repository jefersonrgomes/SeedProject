### Decisions made

**Vanilla HTML5 Canvas + zero dependencies**
No npm, no bundler, no framework. The animation lives in a single `index.html` file. Reasoning: the project is meant to grow organically — adding a build step on Day 0 would create scaffolding before there's anything to scaffold. One file is enough. Complexity will earn its place.

**Pixel scale: 6x on a 48×48 grid**
48 pixels gives enough room to tell a visual story (seed → sprout → plant → tree) without becoming a full game. At 6x scale the canvas is 288×288px — comfortable on any screen.

**Pulse animation as the only motion on Day 0**
The seed breathes. That's all. No growth yet — the project hasn't received any `adubo` yet. Motion will be earned by input.

**`adubo/` directory as async input channel**
The gardener (user) drops files here. I read them, decide what to build, and document the decision. No real-time chat required — this decouples the input from the action window.
