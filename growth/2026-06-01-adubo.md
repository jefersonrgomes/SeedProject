## Decisões arquiteturais

**Por que CDN ESM + import map em vez de bundler?**
O projeto serve `src/` direto no GitHub Pages. Import map (`"three"` →
`three.module.js`, `"three/addons/"` → `examples/jsm/`) dá acesso a Three.js e a
todos os addons (postprocessing) **sem nenhum build**. É o padrão oficial documentado
pela three.js. Mantém a identidade single-file/zero-build do projeto.

**Por que Three.js puro e não Threlte?**
Threlte é Three.js *declarativo para Svelte* — exige compilar `.svelte`. Sem bundler,
vou direto ao Three.js imperativo (que é exatamente o que o Threlte embrulha). Mesma
capacidade, zero build. Spline ficou de fora: o web component precisa de uma cena
`.splinecode` autorada no app proprietário — dependência de asset externo que não quero.

**Por que página separada e não 3D no hero principal?**
Isolamento de risco. WebGL + CDN externo são pontos de falha; o site 2D é a base
estável (zero-dependency, funciona offline-ish). A proposta 3D é um *experimento
acessível por escolha* — degrada sozinha sem afetar o principal. Alinha com
[[feedback_testing]] (não introduzir regressão) e com a nova diretriz de buscar a
melhor experiência visual sem comprometer a estabilidade.
