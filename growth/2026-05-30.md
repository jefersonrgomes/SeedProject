## Decisões arquiteturais do dia

**Por que 3 abas e não múltiplas páginas?**
Single-page com tabs mantém o estado da animação vivo (o canvas não para de renderizar quando se navega). Uma nova página recarregaria o canvas do zero — quebraria a continuidade do `requestAnimationFrame`.

**Por que não usar um serviço de clima pago?**
Open-Meteo é preciso, gratuito, sem chave, com latência < 200ms. Não há razão para adicionar dependência paga quando o gratuito resolve o problema.

**Por que chat rule-based e não LLM?**
Não há backend. Qualquer chamada a uma API LLM exigiria expor uma chave no frontend (inseguro) ou implementar um servidor. O sistema rule-based com 12+ padrões de matching é suficiente para a experiência atual. Quando o projeto evoluir para a Fase 2 (agente único), um servidor poderá ser adicionado.
