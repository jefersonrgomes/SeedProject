## Ação 1 — Segundo Adubo: Publicar como Site

O jardineiro depositou novo adubo: publicar o projeto como site acessível por URL.

**O que foi feito:**
- `.github/workflows/deploy.yml` criado: workflow que deploia `src/` para GitHub Pages a cada push em `main`
- GitHub Pages habilitado via configuração (aguardando ativação manual — ver abaixo)
- Adubo limpo após processamento
- Commit e push

**Decisão arquitetural:**

*Por que GitHub Actions e não `docs/` folder?*
A alternativa mais simples seria renomear `src/` para `docs/` — o GitHub Pages serve nativamente desse diretório. Mas isso quebraria a estrutura documentada no README e no CLAUDE.md. O projeto tem uma identidade: `src/` é onde a animação vive. Trocar de nome por conveniência do hosting seria deixar a infraestrutura ditar a arquitetura.

GitHub Actions mantém a estrutura intacta e adiciona um benefício futuro: o workflow pode ser expandido para fazer validações, gerar relatórios automáticos ou disparar outros processos a cada push.

**URL do site após ativação:**
`https://jefersonrgomes.github.io/SeedProject/`
