## Bug: Canvas não aparecia no hero

**Causa raiz**: `.stage { opacity: 0; }` no CSS combinado com Motion CDN.

O padrão CSS `opacity: 0` + Motion `animate('.stage', {opacity:[0,1]})` funciona assim:
- CSS define o stage como invisível
- Motion CDN é importado (network request)
- **Durante o carregamento do CDN, o canvas fica invisível**
- Quando Motion executa, anima opacity de 0 para 1

Se o CDN demora (rede lenta, cache miss), o usuário vê um hero vazio por até 2-3 segundos.

**Fix**: Removido `opacity: 0` do `.stage` CSS. O canvas é visível **imediatamente** ao carregar.

O Motion ainda faz a animação de entrada: `animate('.stage', {opacity:[0,1], scale:[0.96,1]})` define o keyframe inicial como 0 (mesmo sem CSS) e anima para 1. Se Motion carregar → fade in suave. Se Motion falhar → canvas visível imediatamente. **Win-win**.

**Segunda causa**: `height: auto` numa `<canvas>` não garante proporção quadrada em todos os browsers. Substituído por `height: clamp(300px, min(82vw, 70vh), 580px)` (explícito, igual ao width).

## Melhoria Hero

### Canvas maior
- De `max 520px` para `max 580px`
- Usa `min(82vw, 70vh)` — respeita tanto largura quanto altura do viewport
- Em desktop 1080p: `min(82vw=1574, 70vh=756) → 756px` → clamp → **580px**
- Em mobile 375px: `min(82vw=307, 70vh=467) → 307px` → clamp → **307px**

### Glow radial no background do hero
```css
#v-semente {
  background: radial-gradient(ellipse 70% 55% at 50% 42%, var(--glow-g), transparent 68%);
}
```
O glow radial é centrado no canvas (42% de Y = levemente acima do centro), criando a sensação de que a luz emana do canvas para o fundo.

### Canvas-wrap: glow maior
- `0 0 80px var(--glow-g)` (era 40px)
- `0 0 160px rgba(0,0,0,0.3)` — halo escuro externo para contraste
- Hover: glow 120px para dar feedback ao cursor

### Sem float animation no hero
Removido `animation: float` do `.canvas-wrap` no hero — o canvas grande não precisa oscilar, a animação de vento no pixel art já dá vida suficiente.

### Sidebars compactas
Blocos secundários (stack, skills) ocultos no hero via `:nth-child(n+4)` e `:nth-child(n+3)`. A primeira sessão mostra apenas: sobre, dias de vida, fase, humor, música — o essencial.

## Decisões arquiteturais

**Por que não usar CSS animation como fallback?**
CSS animation e Motion's inline styles conflitariam. A solução mais simples é não esconder o elemento com CSS — deixar a aplicação funcionar sem Motion.

**Por que glow radial no container e não só no canvas-wrap?**
O glow no canvas-wrap é pontual (bordas do elemento). O glow radial no container cria um halo que "sangra" pelo background, dando a sensação de que o canvas irradia luz para o ambiente. Dois efeitos complementares.
