## Fase 2 — CRESCER: broto verde

**Elemento adicionado:** broto acima do solo (`stage >= 3` em `drawSprout()`)

```javascript
if (stage >= 3) {
  const sp = 0.75 + Math.sin(t * 0.03) * 0.15;
  px(cx,   28, P.sprout, sp * 0.9); // na superfície, virando verde
  px(cx,   27, P.sprout, sp);       // o broto! acima do solo
  px(cx-1, 27, P.leaf,   sp * 0.5); // arqueamento lateral
  px(cx+1, 27, P.leaf,   sp * 0.5);
  px(cx-1, 26, P.sprout, sp * 0.65); // cotilédones dobrando
  px(cx+1, 26, P.sprout, sp * 0.65);
  px(cx,   26, P.leaf,   sp * 0.45);
}
```

**Por que `P.sprout = '#68a030'` e não a cor do hipocótilo `'#a0b840'`?**
Com luz solar, a clorofila se forma — o verde se intensifica. O hipocótilo era amarelo-esverdeado (etiolado = sem luz). O broto acima do solo já tem acesso à luz, então fica verde vibrante (`#68a030`). A transição de cor conta a história biologicamente correta.

**`CANVAS_STAGE = 3`** declarado como constante. Será incrementado a cada `/crescer-canvas`.

## Fase 3 — REVISAR: unificação de drawCrack + drawHypocotyl

**Problema identificado:** duas funções separadas (`drawCrack`, `drawHypocotyl`) gerenciavam elementos do mesmo estágio. Violação do princípio de coesão.

**Solução:** `drawSprout(stage)` unificada:
- `stage >= 2`: rachadura + hipocótilo etiolado
- `stage >= 3`: broto verde + cotilédones dobrando
- `stage >= 4`: placeholder para cotilédones abertos (próxima sessão)

Render simplificado:
```javascript
// Antes: drawSoil(); drawCrack(); drawHypocotyl(); ...
// Depois:
drawSoil(); drawSprout(CANVAS_STAGE); drawWorm(); drawRain(); drawHeart();
```

**Resultado:** código mais fácil de evoluir. Adicionar um novo estágio = adicionar um `if (stage >= N)` dentro de `drawSprout`.

## Fase 4 — EVOLUIR: mood-driven CSS variables

**Feature implementada:** quando o humor da semente muda (com base no clima), as variáveis CSS `--accent-hi` e `--gold` mudam sutilmente.

```javascript
const MOOD_THEMES = {
  'Crescimento Radiante': { hi:'#8acc5a', gold:'#e0c468' },
  'Energia Solar Plena':  { hi:'#a0dc4a', gold:'#f0d040' },
  'Alegria da Chuva':     { hi:'#5abccc', gold:'#d8cc68' },
  // ... 10 moods total
};
```

**Efeito visual:** todos os botões, bordas, badges, progress bar, e texto `accent-hi` mudam de cor suavemente conforme o humor. O site "respira" junto com o clima externo.

**Por que CSS variables e não classes?** CSS variables propagam automaticamente por toda a cascade. Mudar uma variável atualiza todos os elementos que a referenciam sem precisar de JS adicional. E os `transition: color 0.2s` já existentes nos elementos fazem a transição visual suave.
